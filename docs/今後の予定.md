## 1. スケジュール確定後のメール送信機能

### 目的
スケジュールが確定した際、申し込み依頼をくれた学校（教員）のメールアドレス宛に「あなたの学校のスケジュールが決まりました」という確定通知を自動送信する。

### 仕様

#### 送信タイミング
- SA画面の「スケジュール確定」ボタンを押したタイミング
- または、確定後に手動でメール送信ボタンを押す形（トリガー方式は要検討）

#### 送信先
- 候補日シートに登録されているメールアドレス（学校代表）
- 兼任SA・支援員にはスタッフ用通知を別途検討

#### メール内容（案）
```
件名: 【FSL鹿児島】◯月分のICT支援スケジュールが確定しました

◯◯学校 ◯◯先生

◯月分のICT支援スケジュールが確定しましたのでお知らせします。

【確定内容】
1回目: 2026-06-15 (月) 09:00-12:00 訪問支援  担当: 鈴木 亮
2回目: 2026-06-22 (月) 13:30-16:30 訪問支援  担当: 鈴木 亮

なお、当日のICT支援要望としていただいた内容は支援員に共有済みです。
ご不明な点があれば、SA担当までご連絡ください。

FSL鹿児島県ICT支援スケジュールシステム
```

### 注意事項（ユーザー指示）
- **本番環境でお客様から「GO」が出るまで実装しない**
- 開発環境/テスト環境ではダミーメールアドレスへの誤送信リスクがあるため、現状は手動連絡で運用
- 本番投入時は「テスト送信モード」（特定アドレスのみ）→「本番送信モード」（全件）の段階解放を検討

### 実装メモ
- GAS の `MailApp.sendEmail()` または `GmailApp.sendEmail()` を使用
- 送信履歴をシート（例: `送信ログ`）に残す
- 重複送信防止のフラグ（送信済み学校）も併せて持つ
- 件名・本文テンプレートはシステム設定シートで編集可能にすると保守性が高い

## 2. カレンダーの仮予約機能

### 目的
学校から「スケジュールに関するコメント」欄で「翌月以降の◯月◯日に来てほしい」という依頼があった場合、SA側でそのコメントを拾って、翌月以降のカレンダー上に仮予約を入れられるようにする。

### 背景
現状の運用では：
- 学校画面の「スケジュールに関するコメント」欄に補足文言を表示
  > ※対象月以降の支援予定を依頼されたい場合は、このコメント欄に記載をお願いします
- 学校が記載した依頼を SA が読み取り、スプレッドシートに直接書き込む運用
- ただし「直接書き込み」は安全性・追跡性の観点で望ましくない

### 仕様（案）

#### SA画面に追加する機能
1. **コメント検出**
   - 候補日のコメント欄に「対象月以降」「来月」「◯月◯日」等のキーワードが含まれる行を自動的にハイライト表示
   - SA画面のダッシュボードに「事前依頼の可能性あり」のバッジ
2. **仮予約の登録**
   - SA画面のスケジュール管理で、対象月以外の月にも切り替え可能にする
   - 切替先の月で、空き枠（未確定）に「仮予約」として支援員・学校・日付・時間帯を登録できる
   - 仮予約は通常のスケジュールエントリと別の「ステータス＝仮予約」で識別
3. **仮予約 → 候補日への昇格**
   - その月の候補日入力受付が始まった時点で、仮予約を該当学校の候補日エントリに自動移行
   - 移行時に教員に「事前依頼として登録された予定です」と通知（メール送信機能と連動）

### 実装メモ
- スケジュールシートに「ステータス」列に新値「仮予約」を追加
- `getSchedule_` の戻り値に「仮予約」フラグを反映
- 自動割当アルゴリズムでは、仮予約済みの (date, slot) も usedSlots に反映して衝突を防ぐ
- カレンダーチップは仮予約は破線枠／薄色などで通常確定と区別

### 留意点（ユーザー指示）
- **今回（2026-05-08時点）は実装しない**
- 仕様検討と将来的な実装に向けて記録のみ
- スプレッドシート直接編集の運用は望ましくないため、最終的にはこの機能で代替したい

## 3. 支援工数レポート出力＋メール送信機能

### 目的
月次で、純粋なICT支援員4名（水本／村永／谷口／鈴木）の **自治体ごとの支援工数（合計時間）** を、社内で定型化された Excel(.xlsx) フォーマットに書き出し、ZIPでまとめて指定のSAへメール添付送信する。送信されたExcelは社内の原価計算システムに読み込まれる。

### 仕様（2026-05-14 確定）

#### 集計対象支援員
- **水本／村永／谷口／鈴木** の4名のみ
- 上記4名は「純粋なICT支援員」として **支援工数** をカウント
- それ以外のSAメンバーは「SA工数」として別管理のため本機能の対象外
- 表記揺れ照合：姓のみ前方一致＋全角/半角スペース正規化（暫定。厳密化が必要になれば再調整）

#### 集計対象データ
- `スケジュール`シート（要望からの自動割当・確定分）
- `手入力スケジュール`シート（要望取得を行わない自治体向け。**こちらの方が件数は多い見込み**）
- **両シートに入っている全予定**が集計対象
- `代行(origStaff)`が入っている日は、**実際に従事した支援員**にカウント（`staffName` 側）

#### 工数換算ルール
| 時間帯 | 工数 |
|---|---|
| `09:00-12:00`（AM）  | 3.0h |
| `13:30-16:30`（PM）  | 3.0h |
| `09:00-16:00`（終日）| 6.0h（休憩1h除く） |
| オンライン（任意時刻） | `HH:MM-HH:MM` 差分から計算 |
| 計算結果の丸め | **15分単位**（0.25 / 0.5 / 0.75）に統一 |

#### 起動方法
- SA画面に新規UI：
  - 対象月セレクタ（`<input type="month">`）
  - 送信先選択（**SAマスタの複数選択チェックボックス**、1〜4名想定）
  - 「工数レポート送信」ボタン
- 月初トリガー自動化は **行わない**（手動実行のみ）

#### 出力構造（ZIP）
```
支援工数_2026-06.zip
├ 鹿児島県/
│   ├ 水本_2026-06.xlsx
│   ├ 村永_2026-06.xlsx
│   └ 谷口_2026-06.xlsx
├ 枕崎市/
│   ├ 村永_2026-06.xlsx
│   └ 鈴木_2026-06.xlsx
…
```
- **1ファイル = 1支援員 × 1自治体 × 1か月**
- ファイル単位は最小化（該当データが0時間なら作らない）

#### Excelの中身
- 形式：**.xlsx（必須）**。CSV不可。社内システムが.xlsxを読む。
- 列構成：`氏名 / 作業番号 / 合計時間 / 単価`
- **明細は不要、合計のみ**
- 作業番号は **自治体単位** で紐付け（自治体マスタに紐づく1対1）
- 単価セルも Excel 内に存在する

#### 単価データの保管
- **専用スプレッドシート**で運用（頻繁に変動するため、編集容易性を優先）
- スクリプトプロパティ案は不採用（更新時の操作性が悪い）
- シート例：`支援員単価マスタ`（姓 / 単価 / 適用開始月 / 備考）
- アクセスは月次レポート生成時の1回のみ。常時アクセスは避ける。

#### メール送信
- `GmailApp.sendEmail()`（添付サイズ余裕あり、想定 < 1MB）
- 送信先：SA画面で選択した複数SA（全員 SAマスタに登録済み）
- 件名・本文テンプレートはコード内ベタで開始、後日設定シート化を検討

### 着手前に提供してもらう必要があるもの
1. **Excelテンプレートファイル**（.xlsx もしくは雛形 Google スプレッドシート）
2. **自治体→作業番号の対応表**（自治体マスタへの列追加 or 別シート）
3. テンプレ内の **書き込みセル位置一覧**（氏名=○、作業番号=○、合計時間=○、単価=○）

### 実装メモ
- **.xlsx 出力の標準パターン**：
  1. 一時的に Google Spreadsheet を `DriveApp.createFile` で作成、テンプレを `copyTo` で複製
  2. 必要セルに値を書き込む（`getRange().setValue()`）
  3. `UrlFetchApp.fetch(exportUrl, { headers: { Authorization: 'Bearer ' + ScriptApp.getOAuthToken() }})` で .xlsx blob を取得
     - URL: `https://docs.google.com/spreadsheets/d/{id}/export?format=xlsx`
  4. 一時スプレッドシートを `DriveApp.getFileById(id).setTrashed(true)` で削除
- **ZIP化**：`Utilities.zip(blobs, "支援工数_yyyy-MM.zip")`
  - 各 blob の name に `"鹿児島県/水本_2026-06.xlsx"` のようにスラッシュ区切りで指定するとZIP内のフォルダ構造として保持される
- **集計クエリ**：
  - 期間：対象月の `01〜末日`
  - スケジュール側：`status === '確定'` のもの ＋ `targetMonth === 対象月`
  - 手入力側：`日付 in [対象月]` ＋ `ステータス !== '取消'`（仕様要確認）
  - 学校 → 自治体のマッピングは学校マスタ（手入力側は既に自治体カラムを持つ）
- **代行ロジック**：`origStaff` 有無に関わらず `staffName` で集計、`origStaff` は参考情報として無視
- **作業番号紐付け**：自治体マスタの拡張 or 別シート（要・テンプレ提供時に決定）
- **アトミック性**：レポート生成中に一時シートが残らないよう、try/finally で削除を保証

### 開発時の注意点
- 単価情報を含むため、**ログ出力に単価を含めない**
- メール本文に金額は含めない（添付Excel内のみ）
- 失敗時はエラーメールが暴露しないよう、エラーメッセージは抽象化
- テスト時は **自分のメールアドレスのみ宛先** にしてサニティチェック

### 想定工数
- テンプレ・作業番号一覧受領後、**2〜4時間** で実装可能
- 関連ファイル：`コード.js`（新規関数群）、`sa.html`（UI追加）
