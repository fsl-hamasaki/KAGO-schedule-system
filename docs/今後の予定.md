## 1. スケジュール確定後のメール送信機能

### 目的
スケジュールが確定した際、申し込み依頼をくれた学校（教員）のメールアドレス宛に「あなたの学校のスケジュールが決まりました」という確定通知を自動送信する。

### 仕様

#### 送信タイミング
- SA画面の「スケジュール確定」ボタンを押したタイミング
- または、確定後に手動でメール送信ボタンを押す形（トリガー方式は要検討）

#### 送信先
- 候補日シートに登録されているメールアドレス（学校代表）
- 兼任SA・支援員にはスタッフ用通知を別途検討

#### メール内容（案）
```
件名: 【FSL鹿児島】◯月分のICT支援スケジュールが確定しました

◯◯学校 ◯◯先生

◯月分のICT支援スケジュールが確定しましたのでお知らせします。

【確定内容】
1回目: 2026-06-15 (月) 09:00-12:00 訪問支援  担当: 鈴木 亮
2回目: 2026-06-22 (月) 13:30-16:30 訪問支援  担当: 鈴木 亮

なお、当日のICT支援要望としていただいた内容は支援員に共有済みです。
ご不明な点があれば、SA担当までご連絡ください。

FSL鹿児島県ICT支援スケジュールシステム
```

### 注意事項（ユーザー指示）
- **本番環境でお客様から「GO」が出るまで実装しない**
- 開発環境/テスト環境ではダミーメールアドレスへの誤送信リスクがあるため、現状は手動連絡で運用
- 本番投入時は「テスト送信モード」（特定アドレスのみ）→「本番送信モード」（全件）の段階解放を検討

### 実装メモ
- GAS の `MailApp.sendEmail()` または `GmailApp.sendEmail()` を使用
- 送信履歴をシート（例: `送信ログ`）に残す
- 重複送信防止のフラグ（送信済み学校）も併せて持つ
- 件名・本文テンプレートはシステム設定シートで編集可能にすると保守性が高い

## 2. カレンダーの仮予約機能

### 目的
学校から「スケジュールに関するコメント」欄で「翌月以降の◯月◯日に来てほしい」という依頼があった場合、SA側でそのコメントを拾って、翌月以降のカレンダー上に仮予約を入れられるようにする。

### 背景
現状の運用では：
- 学校画面の「スケジュールに関するコメント」欄に補足文言を表示
  > ※対象月以降の支援予定を依頼されたい場合は、このコメント欄に記載をお願いします
- 学校が記載した依頼を SA が読み取り、スプレッドシートに直接書き込む運用
- ただし「直接書き込み」は安全性・追跡性の観点で望ましくない

### 仕様（案）

#### SA画面に追加する機能
1. **コメント検出**
   - 候補日のコメント欄に「対象月以降」「来月」「◯月◯日」等のキーワードが含まれる行を自動的にハイライト表示
   - SA画面のダッシュボードに「事前依頼の可能性あり」のバッジ
2. **仮予約の登録**
   - SA画面のスケジュール管理で、対象月以外の月にも切り替え可能にする
   - 切替先の月で、空き枠（未確定）に「仮予約」として支援員・学校・日付・時間帯を登録できる
   - 仮予約は通常のスケジュールエントリと別の「ステータス＝仮予約」で識別
3. **仮予約 → 候補日への昇格**
   - その月の候補日入力受付が始まった時点で、仮予約を該当学校の候補日エントリに自動移行
   - 移行時に教員に「事前依頼として登録された予定です」と通知（メール送信機能と連動）

### 実装メモ
- スケジュールシートに「ステータス」列に新値「仮予約」を追加
- `getSchedule_` の戻り値に「仮予約」フラグを反映
- 自動割当アルゴリズムでは、仮予約済みの (date, slot) も usedSlots に反映して衝突を防ぐ
- カレンダーチップは仮予約は破線枠／薄色などで通常確定と区別

### 留意点（ユーザー指示）
- **今回（2026-05-08時点）は実装しない**
- 仕様検討と将来的な実装に向けて記録のみ
- スプレッドシート直接編集の運用は望ましくないため、最終的にはこの機能で代替したい
